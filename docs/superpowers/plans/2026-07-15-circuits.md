## Global Constraints

- **Toolchain (MUST be installed by Task 1):** the **`circom` compiler** is a Rust binary (`cargo install --git https://github.com/iden3/circom --tag v2.1.6`), NOT an npm package — `circom_tester` shells out to it. **`snarkjs`** must be a declared devDependency (invoked as `npx snarkjs`). Task 1 installs/verifies both before the gate.
- **Poseidon parity (MUST, Task 1 gate):** circom hashes MUST be byte-identical to the on-chain hash. On-chain uses `solana-poseidon` `Parameters::Bn254X5`, `Endianness::BigEndian`. Task 1 proves **BOTH** widths with fixtures: `Poseidon(2)` vs `pool_program::poseidon::hash2`, AND **`Poseidon(1)` vs `solana_poseidon::hashv(Bn254X5, BE, &[x])`** (single input — needed for `nullifierHash`; the on-chain `hash2` is 2-input only and cannot produce it). If either differs, STOP.
- **Canonical Rust hashes (single source, used by all fixture/prover code):**
  - `commitment(nullifier, secret) = pool_program::poseidon::hash2(&nullifier, &secret)`
  - `nullifier_hash(nullifier)     = solana_poseidon::hashv(Bn254X5, BigEndian, &[&nullifier]).to_bytes()`
- **Merkle parity (MUST):** `TREE_HEIGHT = 20`; zero-leaf = field element `0` = `[0u8;32]`. The circom Merkle path uses `hash2(left,right)` with the SAME index-parity ordering as `merkle::insert` (bit 0 ⇒ current is left, sibling right; bit 1 ⇒ sibling left, current right). *(Review verified the plan's `MerkleLevel` already matches — keep it.)*
- **Field encoding:** every value crossing the Rust↔circom↔on-chain boundary is a BN254 field element serialized **big-endian, 32 bytes**, `< BN254_MODULUS`. **In Rust, convert with `Fr::from_be_bytes_mod_order` / explicit byte-reversal — NOT ark's `CanonicalSerialize`/`Deserialize`, which are little-endian** (silent-wrong-value trap). In JS, `BigInt("0x"+beHex)` ↔ `n.toString(16).padStart(64,"0")` is the correct BE round-trip (review-verified).
- **Curve/proof:** Groth16 over BN254 (alt_bn128). For `groth16-solana`: emit G1/G2/public-input bytes **big-endian** (reverse ark's LE output), and **negate the proof's G1 `A` point** (the standard on-chain check negates proof.A, not the VK). Verify against groth16-solana's README at impl time.
- **Commitment / nullifier scheme (this plan fixes; Plan 3 SDK MUST match):** note = `(nullifier, secret)`; `commitment = Poseidon(nullifier, secret)`; `nullifierHash = Poseidon(nullifier)`; membership `MerkleProof(commitment, pathElements[20], pathIndices[20]) == root`. *(Diverges from spec §4's `H(amount,mint,secret,nullifier_seed)` — intentional: amounts/value-conservation are a later circuit; the on-chain leaf is an opaque `[u8;32]`. Plan 3's SDK must build commitments this same 2-input way.)*
- **Note bundle artifact (single source for Tasks 2–5):** the fixture bin emits ONE committed `circuits/test/withdraw_vectors.json` = `{nullifier, secret, commitment, nullifierHash, root, pathElements[20], pathIndices[20]}` for a real note in the real on-chain tree (see Task 2). Every downstream test (JS + Rust) reads it — no per-task re-derivation.
- **Trusted setup:** a small **committed dev `.ptau`** only (labelled dev-only). The dev zkey/VK are produced **deterministically** (`zkey beacon`, fixed beacon) and are the committed artifact of record. The **production multi-party ceremony is deferred to hardening (spec §5)**; the dev ptau MUST NOT ship a release VK.
- **arkworks major coexistence:** the workspace already resolves `ark-bn254 0.4` (via `solana-poseidon → light-poseidon`); the prover's `ark-* 0.5` coexists as a separate major because the prover never links `pool-program` types. **Do NOT bump `ark-circom` to 0.6** (needs arkworks 0.6 + edition2024 + wasmer 6-rc, splitting the arkworks major and breaking groth16-solana 0.2's arkworks-0.5 interop).
```rust
use pool_program::poseidon::hash2;
use solana_poseidon::{hashv, Endianness, Parameters};

fn fe(bytes: [u8; 32]) -> [u8; 32] { bytes }
fn feb(n: u8) -> [u8; 32] { let mut b = [0u8; 32]; b[31] = n; b }
fn hex(b: &[u8; 32]) -> String { b.iter().map(|x| format!("{x:02x}")).collect() }

/// Canonical nullifier hash — single-input Poseidon, same params as hash2.
fn nullifier_hash(nullifier: &[u8; 32]) -> [u8; 32] {
    hashv(Parameters::Bn254X5, Endianness::BigEndian, &[nullifier.as_slice()])
        .expect("in-field")
        .to_bytes()
}

fn main() {
    let args: Vec<String> = std::env::args().collect();
    match args.get(1).map(String::as_str) {
        Some("poseidon-vectors") | None => {
            // include a high-order-byte case so any future BE/LE regression is caught.
            let mut hi = [0u8; 32]; hi[0] = 0x12; hi[1] = 0x34; hi[31] = 0x56;
            let two = [(feb(1), feb(2)), (feb(0), feb(0)), (fe(hi), feb(9))];
            let one = [feb(1), fe(hi), feb(7)];
            let mut out = String::from("{\"poseidon2\":[");
            for (i, (a, b)) in two.iter().enumerate() {
                if i > 0 { out.push(','); }
                out.push_str(&format!("{{\"a\":\"{}\",\"b\":\"{}\",\"h\":\"{}\"}}",
                    hex(a), hex(b), hex(&hash2(a, b).expect("in-field"))));
            }
            out.push_str("],\"poseidon1\":[");
            for (i, x) in one.iter().enumerate() {
                if i > 0 { out.push(','); }
                out.push_str(&format!("{{\"x\":\"{}\",\"h\":\"{}\"}}", hex(x), hex(&nullifier_hash(x))));
            }
            out.push_str("]}");
            println!("{out}");
        }
        Some(other) => { eprintln!("unknown subcommand: {other}"); std::process::exit(2); }
    }
}
```

> **Fixture correctness (review HIGH):** the on-chain tree is a Tornado-style *incremental* tree — later inserts overwrite `filled_subtrees`, so a leaf's insert-time right-siblings are only its authentic path against the root **right after that leaf's insert**. We therefore emit the bundle for the **last-inserted** leaf, computing its path from the `filled_subtrees` snapshot taken **before** its insert, and take the root from the real `insert` call (authoritative — no hash-loop reimplementation).

- [ ] **Step 1: Add the `note-bundle` subcommand**

Extend `crates/parity-fixtures/src/main.rs` (`match` arm `Some("note-bundle")`). It inserts a decoy leaf (so the note lands at index 1, exercising the right-child branch), snapshots `filled_subtrees` before the note's insert, derives the path from that snapshot, and takes the root from `insert`:
```rust
use pool_program::merkle::{empty_root, insert, zeros, TREE_HEIGHT};
// ...
Some("note-bundle") => {
    let z = zeros().expect("zeros");
    let mut next_index = 0u32;
    let mut root = empty_root(&z).expect("empty_root");
    let mut filled = z;

    // decoy leaf at index 0 (so the real note is index 1 -> right child at level 0)
    let decoy = hash2(&feb(111), &feb(222)).expect("in-field");
    insert(&mut next_index, &mut root, &mut filled, decoy).expect("insert decoy");

    // real note
    let nullifier = feb(7);
    let secret = feb(9);
    let commitment = hash2(&nullifier, &secret).expect("in-field");

    // path for the note's index from the snapshot BEFORE its insert
    let filled_before = filled;
    let target = next_index; // 1
    let mut path_elements = [[0u8; 32]; TREE_HEIGHT];
    let mut path_indices = [0u8; TREE_HEIGHT];
    let mut idx = target;
    for i in 0..TREE_HEIGHT {
        let bit = (idx % 2) as u8;
        path_indices[i] = bit;
        path_elements[i] = if bit == 0 { z[i] } else { filled_before[i] };
        idx /= 2;
    }
    // authoritative root from the real insert
    insert(&mut next_index, &mut root, &mut filled, commitment).expect("insert note");

    let nh = nullifier_hash(&nullifier);
    let pe = path_elements.iter().map(hex).collect::<Vec<_>>().join("\",\"");
    let pi = path_indices.iter().map(|b| b.to_string()).collect::<Vec<_>>().join(",");
    println!(
        "{{\"nullifier\":\"{}\",\"secret\":\"{}\",\"commitment\":\"{}\",\"nullifierHash\":\"{}\",\"root\":\"{}\",\"pathElements\":[\"{}\"],\"pathIndices\":[{}]}}",
        hex(&nullifier), hex(&secret), hex(&commitment), hex(&nh), hex(&root), pe, pi
    );
}
```

```circom
pragma circom 2.1.6;
include "poseidon.circom";

template MerkleLevel() {
    signal input cur; signal input sibling; signal input index; signal output out;
    index * (index - 1) === 0;                       // boolean
    signal left; signal right;
    left  <== cur + index * (sibling - cur);         // index==0 ? cur : sibling
    right <== sibling + index * (cur - sibling);      // index==0 ? sibling : cur
    component h = Poseidon(2); h.inputs[0] <== left; h.inputs[1] <== right; out <== h.out;
}

template MerkleProof(depth) {
    signal input leaf; signal input pathElements[depth]; signal input pathIndices[depth];
    signal output root;
    component levels[depth]; signal cur[depth + 1]; cur[0] <== leaf;
    for (var i = 0; i < depth; i++) {
        levels[i] = MerkleLevel();
        levels[i].cur <== cur[i];
        levels[i].sibling <== pathElements[i];
        levels[i].index <== pathIndices[i];
        cur[i + 1] <== levels[i].out;
    }
    root <== cur[depth];
}
component main = MerkleProof(20);
```
