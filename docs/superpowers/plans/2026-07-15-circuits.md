# circuits Implementation Plan (Groth16 membership + nullifier)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Groth16 **withdraw/spend circuit** (Merkle membership + nullifier derivation) in circom, its trusted setup and exported verifier key, and a Rust (`ark-circom`) proving harness — all producing **byte-identical** Poseidon/Merkle results to the on-chain `pool-program` modules, so Plan 3 can wire on-chain verification.

**Architecture:** A `circuits/` directory (circom sources + circomlib Poseidon/Merkle templates + `circom_tester` JS unit tests + snarkjs setup) and a Rust `crates/prover` crate (`ark-circom` proof generation, `groth16-solana`-compatible verification). This is subsystem 2 of Phase 1 (see spec §2); it depends on the finished `poseidon` + `merkle` on-chain modules and is consumed by Plan 3 (wire-ZK).

**Tech Stack:** circom 2.x · circomlib · snarkjs · Node 20+ · Rust (`ark-circom`/`ark-bn254`/`ark-groth16`) · `groth16-solana`.

**Design spec:** [`docs/superpowers/specs/2026-07-15-mirror-pool-design.md`](../specs/2026-07-15-mirror-pool-design.md) · **Depends on:** [`2026-07-15-pool-program-foundations.md`](./2026-07-15-pool-program-foundations.md)

## Global Constraints

- **Poseidon parity (MUST):** the circom hash MUST produce output byte-identical to the on-chain `pool_program::poseidon::hash2` (which uses `solana-poseidon` `Parameters::Bn254X5`, big-endian). `solana-poseidon`'s host path builds `light_poseidon::Poseidon::<Fr>::new_circom(width)`, which is *designed* to match circomlib's `Poseidon`. **Task 1 proves this with a fixture before anything else is built.** If they differ, STOP — do not build the circuit on a mismatched hash.
- **Merkle parity (MUST):** `TREE_HEIGHT = 20`; zero-leaf = field element `0`. The circom Merkle path MUST use the same `hash2(left, right)` and the same left/right ordering the on-chain `merkle::insert` uses (index-parity: bit 0 ⇒ current is left / sibling right; bit 1 ⇒ sibling left / current right).
- **Field encoding:** every value crossing the Rust↔circom boundary is a BN254 field element, serialized **big-endian, 32 bytes**, `< BN254_MODULUS`. circom emits field elements (bigints, little-endian internally) — convert explicitly and test the round-trip.
- **Curve/proof:** Groth16 over **BN254 (alt_bn128)**; the exported verifier key + proof must be in the byte layout `groth16-solana` consumes (uncompressed G1/G2, the negated/formatted VK that library expects — verify against its README at implementation time).
- **Commitment / nullifier scheme (this plan fixes these; Plan 3 + SDK must match):**
  - note secret = two field elements `(nullifier, secret)`
  - `commitment      = Poseidon(nullifier, secret)`   (the leaf inserted on-chain)
  - `nullifierHash   = Poseidon(nullifier)`            (public; the on-chain nullifier PDA seed)
  - membership: `MerkleProof(commitment, pathElements[20], pathIndices[20]) == root`
- **Trusted setup:** this plan uses a **small committed test `.ptau`** (dev only, clearly labelled) for CI/tests. The **production multi-party ceremony is deferred to hardening (spec §5)** — the dev ptau MUST NOT be used for a release VK.
- **Value conservation / amounts (DEFERRED):** this first circuit is membership+nullifier only (Tornado-style; amounts handled by pool denomination). The 2-in/2-out value-conservation circuit is a later plan.
- Node tests: `npm test` in `circuits/` green; Rust: `cargo test -p prover` green.

```rust
// Prints known-answer vectors from the on-chain hash so the circom side can be
// checked against them. inputs are big-endian 32-byte field elements.
use pool_program::poseidon::hash2;

fn fe(n: u8) -> [u8; 32] { let mut b = [0u8; 32]; b[31] = n; b }

fn hex(b: &[u8; 32]) -> String { b.iter().map(|x| format!("{x:02x}")).collect() }

fn main() {
    let cases = [(fe(1), fe(2)), (fe(0), fe(0)), (fe(7), fe(9))];
    let mut out = String::from("[");
    for (i, (a, b)) in cases.iter().enumerate() {
        let h = hash2(a, b).expect("in-field");
        if i > 0 { out.push(','); }
        out.push_str(&format!(
            "{{\"a\":\"{}\",\"b\":\"{}\",\"h\":\"{}\"}}",
            hex(a), hex(b), hex(&h)
        ));
    }
    out.push(']');
    println!("{out}");
}
```

`circuits/circom/merkle_proof.circom`:
```circom
pragma circom 2.1.6;
include "poseidon.circom";

// One level: given current hash and sibling, order by index bit, hash.
template MerkleLevel() {
    signal input cur;
    signal input sibling;
    signal input index; // 0 => cur is left; 1 => cur is right
    signal output out;

    index * (index - 1) === 0; // boolean

    // left  = index==0 ? cur : sibling ; right = index==0 ? sibling : cur
    signal left;  signal right;
    left  <== cur + index * (sibling - cur);
    right <== sibling + index * (cur - sibling);

    component h = Poseidon(2);
    h.inputs[0] <== left;
    h.inputs[1] <== right;
    out <== h.out;
}

template MerkleProof(depth) {
    signal input leaf;
    signal input pathElements[depth];
    signal input pathIndices[depth];
    signal output root;

    component levels[depth];
    signal cur[depth + 1];
    cur[0] <== leaf;
    for (var i = 0; i < depth; i++) {
        levels[i] = MerkleLevel();
        levels[i].cur <== cur[i];
        levels[i].sibling <== pathElements[i];
        levels[i].index <== pathIndices[i];
        cur[i + 1] <== levels[i].out;
    }
    root <== cur[depth];
}
component main = MerkleProof(20);
```

`circuits/circom/withdraw.circom`:
```circom
pragma circom 2.1.6;
include "poseidon.circom";
include "merkle_proof.circom";

template Withdraw(depth) {
    // public
    signal input root;
    signal input nullifierHash;
    // private
    signal input nullifier;
    signal input secret;
    signal input pathElements[depth];
    signal input pathIndices[depth];

    // commitment = Poseidon(nullifier, secret)
    component cm = Poseidon(2);
    cm.inputs[0] <== nullifier;
    cm.inputs[1] <== secret;

    // nullifierHash = Poseidon(nullifier) — bind public input
    component nh = Poseidon(1);
    nh.inputs[0] <== nullifier;
    nh.out === nullifierHash;

    // membership: recomputed root must equal the public root
    component mp = MerkleProof(depth);
    mp.leaf <== cm.out;
    for (var i = 0; i < depth; i++) {
        mp.pathElements[i] <== pathElements[i];
        mp.pathIndices[i] <== pathIndices[i];
    }
    mp.root === root;
}
component main {public [root, nullifierHash]} = Withdraw(20);
```

## What this plan delivers

A circom **withdraw circuit** (Merkle membership + nullifier), proven byte-compatible with the on-chain `poseidon`/`merkle` modules via committed cross-language fixtures; a reproducible dev trusted setup + exported `verification_key.json`; and a Rust `ark-circom` proving harness with an end-to-end prove→verify test. Plan 3 consumes the VK (embeds it in `pool-program`) and the prover (in the SDK).

## Explicitly deferred to later plans

- **On-chain proof verification** (`commit_intent`/`withdraw` in `pool-program` verifying this VK, spending the nullifier) — Plan 3 (wire-ZK).
- **Production trusted-setup ceremony** (multi-party, real entropy) — hardening (spec §5). The dev ptau/zkey here are labelled dev-only and MUST NOT ship.
- **Value-conservation / amounts circuit** (2-in/2-out UTXO) — a later plan; this circuit is membership+nullifier only.
- **VK-into-program codegen + `check-vk` CI guard** — Plan 3 + the CI doc's e2e job.

## Open questions / risks

- **Poseidon parity (Task 1) is the gating risk.** Strong prior (`light-poseidon new_circom` targets circomlib), but unproven until the fixture is green. Everything downstream assumes it holds.
- **"Rust only" eligibility** — circom DSL + snarkjs setup are not Rust (user-accepted trade-off; see top note).
- **`ark-circom` / `groth16-solana` versions & VK byte layout** — pin and verify against their current READMEs at implementation time (the `groth16-solana` VK format has specific G1/G2 negation/endianness requirements).
- **ptau power** — confirm the withdraw circuit's constraint count fits the chosen ptau (`2^14`); bump if the height-20 Merkle path pushes it over.
