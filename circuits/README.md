# circuits/

The zero-knowledge circuits — a circom Groth16 proof of note membership, over the *same*
Poseidon/Merkle hashes the on-chain program uses.

## What the `withdraw` circuit proves

Given the private witnesses (a note's `secret`/`nullifier` and its Merkle path), it proves three
things without revealing which note:

1. the note's commitment is a **leaf under a tree with the public `root`**;
2. the public **`nullifierHash` = Poseidon(nullifier)** (so the on-chain single-spend check binds
   to *this* note); and
3. it exposes a public **`extDataHash`** binding the payout recipient/relayer/fee — so the proof
   is worthless to a relayer trying to redirect funds.

**Public inputs (fixed order): `[root, nullifierHash, extDataHash]`** — 32-byte big-endian each.
The on-chain program additionally enforces that `root` is a *known recent* root and that the
nullifier is unspent; those are the program's job, not the circuit's.

## Files

```
circom/withdraw.circom          the main circuit (membership + nullifier + extDataHash)
circom/merkle_proof*.circom     depth-20 Merkle inclusion
circom/poseidon1.circom         Poseidon width-1 (nullifierHash)
circom/poseidon2.circom         Poseidon width-2 (commitment + Merkle nodes)
scripts/setup.sh                deterministic dev Groth16 setup (ptau → zkey → VK)
test/                           circom↔Rust parity tests + fixed vectors
ptau/                           powers-of-tau (see ptau/README.md)
```

## Poseidon/Merkle parity

The proof is only meaningful if the circuit hashes *exactly* match the chain's. The `test/`
suite (`poseidon_parity`, `merkle_parity`, `withdraw`) pins this: circom's Poseidon(1)/Poseidon(2)
and the Merkle root are asserted **byte-identical** to `pool-program`'s `sol_poseidon`/`merkle`
output, via fixtures generated by [`crates/parity-fixtures`](../crates/parity-fixtures). Any drift
here would silently break every honest proof.

## Trusted setup — dev-only

`scripts/setup.sh` runs a **deterministic** Groth16 setup (a fixed, publicly-known beacon;
sha256-pinned ptau). **This is a development artifact: its toxic waste is public, so it is
forgeable and must NOT be used in production.** A real multi-party ceremony is required before
mainnet. The resulting verifying key is codegen'd into the on-chain program by
[`crates/vk-gen`](../crates/vk-gen) (with a CI drift-guard).

## Build

Requires **circom 2.1.6** (the Rust compiler binary, not the npm package) and **snarkjs**
(`npm ci` here installs snarkjs + circomlib).

```bash
npm ci
bash scripts/setup.sh     # compile, setup, export the VK
npm test                  # the parity + witness tests
```
